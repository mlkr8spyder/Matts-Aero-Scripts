# Simulink Model Setup Instructions

**Target:** MATLAB R2025b with Simulink

## Prerequisites

### Required Toolboxes

| Toolbox | Purpose | Required? |
|---------|---------|-----------|
| MATLAB | Core language | Yes |
| Simulink | Model simulation | Yes |
| Simulink Test | Model validation | Recommended |
| Stateflow | Advanced state machines (future) | Optional |

### Required Data Files

These are generated by the Python pipeline. If not already present, run:

```bash
cd Gauging/synthetic_fuel_system
python -m src.hv_table_generator       # → data/tank_system.mat
python -m src.simulate_sequences        # → data/defuel_sequence.mat, refuel_sequence.mat
```

Verify these files exist:
```
data/tank_system.mat          # H-V tables (1,105 tables, ~586 KB)
data/defuel_sequence.mat      # Defuel time history (1000 samples)
data/refuel_sequence.mat      # Refuel time history (800 samples)
```

## Step-by-Step Setup

### Step 1: Open MATLAB and Navigate

```matlab
cd('path/to/Gauging/synthetic_fuel_system/matlab')
```

### Step 2: Create the MATLAB Project

```matlab
run('scripts/setup_project.m')
```

**What this does:**
- Creates directory structure (`models/`, `scripts/`, `tests/`, `utilities/`, `work/`)
- Creates `FuelGaugingProject.prj` with managed paths
- Creates `startup.m` and `shutdown.m` hooks
- Creates empty `data/FuelGaugingData.sldd` data dictionary
- Sets up label categories for file organization

**Expected output:**
```
=== Fuel Gauging System — Project Setup ===
Project root: .../synthetic_fuel_system
  Created Classification labels
  Created Component labels
  Created: startup.m
  Created: shutdown.m
  Created data dictionary: .../FuelGaugingData.sldd
=== Project Setup Complete ===
```

### Step 3: Populate the Data Dictionary

```matlab
run('scripts/build_data_dictionary.m')
```

**What this does:**
- Creates 6 Simulink bus definitions (AttitudeBus, ProbeReadingBus, etc.)
- Creates 24 Simulink.Parameter objects (thresholds, physical constants)
- Creates TankParams structure (5-tank geometry and probe data)
- Builds 5 Simulink.LookupTable objects (3-D H-V tables from Python data)
- Saves everything to `FuelGaugingData.sldd`

**Expected output:**
```
=== Build Data Dictionary ===
--- Creating Bus Definitions ---
  AttitudeBus
  ProbeReadingBus
  ...
--- Creating System Parameters ---
  Created 24 parameters
--- Creating Lookup Table Objects ---
  LUT_T1: 37×13×17 = 8177 entries
  LUT_T2: 39×13×17 = 8619 entries
  ...
=== Data Dictionary Build Complete ===
```

### Step 4: Build the Simulink Models

```matlab
run('scripts/build_simulink_model.m')
```

**What this does:**
- Creates 8 Simulink model files (`.slx`) in `models/`:

| Model | Type | Description |
|-------|------|-------------|
| `FuelGaugingSystem.slx` | Top-level harness | Integrates all subsystems |
| `ProbeModel_Real.slx` | Referenced model | Single capacitance probe (T1, T2, T4) |
| `ProbeModel_Combo.slx` | Referenced model | Dual-probe with blend zone (T3) |
| `ProbeModel_Pseudo.slx` | Referenced model | Trigonometric projection (T5) |
| `ProbeFailureDetector.slx` | Referenced model | Open/short/rate failure detection |
| `HV_TableLookup.slx` | Referenced model | 3-D n-D Lookup Table |
| `DensityComputation.slx` | Referenced model | Dielectric → density |
| `RefuelSystem.slx` | Referenced model | Single-point refuel + shutoff |

**Expected output:**
```
=== Build Simulink Model (v2) ===
Building ProbeModel_Real...
  Saved: ProbeModel_Real.slx
Building ProbeModel_Combo...
  Saved: ProbeModel_Combo.slx
...
Building FuelGaugingSystem (top level)...
  Saved: FuelGaugingSystem.slx
=== Simulink Model Build Complete ===
```

### Step 5: Validate

```matlab

```

**What this does:**
- Verifies all 8 `.slx` files exist
- Checks data dictionary contains all required entries (buses, params, LUTs)
- Validates lookup table dimensions and monotonicity
- Verifies bus element counts
- Attempts to load each model

## Running Simulations

### Defuel Scenario

```matlab
% Load defuel data
[sim_data, meta] = load_sequence_data('defuel');

% Open and configure model
open_system('FuelGaugingSystem');
set_param('FuelGaugingSystem', 'StopTime', num2str(meta.duration_s));

% Replace constant inputs with From Workspace blocks (manual step)
% Or run programmatically:
out = sim('FuelGaugingSystem');

% Extract results
total_wt = out.total_fuel_weight;
logsout = out.logsout;
```

### Refuel Scenario

```matlab
% Load refuel data
[sim_data, meta] = load_sequence_data('refuel');

% Set refuel command to 1
set_param('FuelGaugingSystem/Refuel_Cmd', 'Value', '1');
set_param('FuelGaugingSystem', 'StopTime', num2str(meta.duration_s));

out = sim('FuelGaugingSystem');
```

## Architecture Overview

### Data Flow

```
Fuel WL inputs (per tank)
         │
         ▼
┌─────────────────┐   ┌────────────────┐   ┌───────────────┐
│  Probe Models   │──▶│ Failure Detect  │──▶│ H-V Lookup    │
│  (Real/Combo/   │   │ (open/short/    │   │ (3-D n-D LUT  │
│   Pseudo)       │   │  rate check)    │   │  from .sldd)  │
└─────────────────┘   └────────────────┘   └───────┬───────┘
                                                    │ volume_in3
                                                    ▼
                      ┌────────────────┐   ┌───────────────┐
Dielectric input ────▶│ Density Comp.  │──▶│ Weight = V×ρ  │──▶ Total Weight
                      │ (model + bias) │   │ (per tank)    │
                      └────────────────┘   └───────────────┘

Refuel cmd ──────────▶ RefuelSystem
Fuel WL inputs ──────▶ (HLS + valves + all-full detect) ──▶ Refuel Status
```

### Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| **Data dictionary** instead of base workspace | Reproducibility; no workspace pollution; version-trackable |
| **Model references** instead of subsystems | Independent compilation; reuse across instances; cleaner hierarchy |
| **3-D n-D Lookup Tables** instead of simplified gain | Accurate volume at all attitudes; matches Python pipeline exactly |
| **Bus objects** for inter-model data | Type safety; self-documenting interfaces; signal logging clarity |
| **Instance-specific model arguments** | One ProbeModel_Real reused for T1/T2/T4 with different probe heights |
| **Autorouting** on all lines | Clean diagram layout without manual positioning |

### Modifying Parameters

All parameters live in the data dictionary. To change a value:

```matlab
dd = Simulink.data.dictionary.open('data/FuelGaugingData.sldd');
ddata = getSection(dd, 'Design Data');

% Example: change density bias from 0.3% to 0.5%
entry = getEntry(ddata, 'DENSITY_BIAS');
p = getValue(entry);
p.Value = 1.005;
setValue(entry, p);

saveChanges(dd);
close(dd);
```

### Adding a New Tank

1. Add tank geometry to `TankParams` in `build_data_dictionary.m`
2. Generate H-V tables for the new tank (Python pipeline or `generate_hv_tables.m`)
3. Add `LUT_T6` lookup table object to the dictionary
4. Add probe/failure/lookup chain in `build_simulink_model.m`
5. Update bus dimensions (e.g., `BIT_StatusBus.failure_flags` from `[1×5]` to `[1×6]`)

## Troubleshooting

### "Data dictionary not found"
Run `setup_project.m` first, then `build_data_dictionary.m`.

### "Lookup table object not found"
The H-V table data (`data/tank_system.mat`) must exist before running `build_data_dictionary.m`. Generate it with:
```bash
python -m src.hv_table_generator
```

### "Model reference cannot be found"
Ensure `models/` is on the MATLAB path. The project setup handles this automatically when the project is open. If running outside the project:
```matlab
addpath('models');
```

### "Bus type mismatch"
Rebuild the data dictionary (`build_data_dictionary.m`) to ensure bus definitions match the model expectations. All buses are defined in one place — the dictionary — so conflicts are eliminated.

### Simulation is slow
- Check `MaxStep` (default 0.1s). Increase to 0.5s for faster runs when valve dynamics are not critical.
- Build model reference simulation targets: `slbuild('ProbeModel_Real', 'ModelReferenceSimTarget')`
