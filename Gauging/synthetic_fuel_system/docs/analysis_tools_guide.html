Introduction

The analysis/ package provides geometry-agnostic tools for importing, interpolating, comparing, and visualizing fuel gauging H-V table data. It works with both the synthetic simulation and real aircraft data (15+ tanks, irregular geometry, variable-length tables).

This guide walks you through the full workflow from data preparation to dashboard generation. For the synthetic system reference, see the Fuel System Guide.

Data Format Specifications

MM1D .mat Files

The .mat importer supports two layouts:

Flat layout (most common): One MATLAB variable per field.

# Required variables:
pitch_range          — [1 × N_pitch] vector of pitch angles (degrees)
roll_range           — [1 × N_roll] vector of roll angles (degrees)
{TankID}_heights     — {N_pitch × N_roll} cell array, each cell is a 1-D vector
{TankID}_volumes     — {N_pitch × N_roll} cell array, matching heights

# Optional:
{TankID}_cg_fs/bl/wl — {N_pitch × N_roll} cell arrays for CG coordinates
{TankID}_fs_min, _fs_max, _bl_min, _bl_max, _wl_min, _wl_max — geometry

Nested struct layout: A top-level struct with per-tank sub-structs.

Tanks.T1.heights    — cell array
Tanks.T1.volumes    — cell array
Cell arrays are variable-length by definition. Each cell at a different attitude can have a different number of height/volume points. This is the MM1D convention.

Slice CSV Files

One CSV per table at one attitude. Minimum columns: height and volume.

# Example: T3_p2.0_r-4.0.csv
height,volume
0.0,0.0
0.5,2100.0
1.0,4200.0
...

The filename encodes pitch and roll via a regex pattern (default: .*_p(?P<pitch>...)_r(?P<roll>...).csv). Use load_slice_directory() to bulk-load an entire folder.

Test Data CSV

One row per timestep. Scale weights are sparse (NaN except at checkpoints).

# Required columns:
time_s, pitch_deg, roll_deg

# Per-tank (auto-detected by prefix):
indicated_weight_lb_T1, indicated_weight_lb_T2, ...
indicated_volume_gal_T1, indicated_volume_gal_T2, ...

# Optional:
scale_gross_weight_lb    — gross weight from scales (NaN between checkpoints)
dry_weight_lb            — aircraft dry weight
density_system           — system-measured fuel density (lb/gal)

Workflow

Prepare Data
Export H-V tables from your program's database to .mat (flat layout) or CSV slices. Export calibration test data as CSV with the column names above. Customize column names via kwargs if your program uses different names.
Import
from analysis.importers.mat_importer import load_mat_tables
from analysis.importers.csv_importer import load_test_csv

# Load H-V tables
data = load_mat_tables("my_program.mat")

# Load test sequence
test = load_test_csv("defuel_run_042.csv", sequence_type="defuel")
data.test_data.append(test)
Validate
Check that the imported data looks correct: number of tanks, attitudes, height ranges, volume ranges. 
for tid in data.tank_ids:
    tank = data.tanks[tid]
    print(f"{tid}: {tank.n_attitudes} attitudes, max vol = {tank.max_volume:.0f}")
Compare
Compute residuals between indicated and scale reference weights. 
from analysis.comparison import compute_residuals, compute_error_statistics

residuals = compute_residuals(test)
stats = compute_error_statistics(residuals)
print(f"RMS error: {stats['rms_error_lb']:.2f} lb")
Generate Dashboard
Create a self-contained HTML file with interactive plots. 
from analysis.dashboard import generate_dashboard

generate_dashboard(data, residuals, "my_analysis.html")
Open the file in any web browser — no server needed.

Writing Custom Adapters

If your program uses non-standard column names or .mat variable names, pass custom key names as kwargs:

# Custom .mat key names
data = load_mat_tables("program_x.mat",
                       heights_key="H",
                       volumes_key="V",
                       pitch_key="alpha",
                       roll_key="phi")

# Custom CSV column names
test = load_test_csv("test_data.csv",
                     time_col="timestamp",
                     pitch_col="theta",
                     roll_col="phi",
                     indicated_weight_prefix="fqi_wt_",
                     scale_col="platform_gross_wt")

For complex transformations (e.g., converting units, merging multiple source files), write a thin wrapper function that calls the base importers and post-processes the result:

def load_my_program(mat_path, test_csv_path):
    data = load_mat_tables(mat_path, heights_key="H_in", volumes_key="V_cc")

    # Convert cm³ to in³
    for tank in data.tanks.values():
        for table in tank.tables.values():
            table.volumes *= 0.0610237  # cc to in³

    test = load_test_csv(test_csv_path, ...)
    data.test_data.append(test)
    return data

API Reference

Data Containers (analysis/importers/base.py)

ClassPurposeKey Fields
HVTableOne H-V curve at one attitude heights, volumes, pitch_deg, roll_deg, optional CG arrays
TankDataAll tables for one tank tank_id (str), tables dict keyed by (pitch, roll), name, metadata
TestRecordOne timestamped measurement time_s, indicated_weights, scale_weight_lb, density_system
TestDatasetCollection of test records records, sequence_type, tank_ids
FuelSystemDataTop-level container tanks dict, test_data list, config dict

Importers

FunctionPurpose
load_mat_tables(filepath, **kwargs)Load .mat H-V tables (auto-detects flat/nested layout)
load_slice_csv(filepath, ...)Load one CSV slice as an HVTable
load_slice_directory(directory, tank_id, ...)Bulk-load CSV slices for one tank
load_test_csv(filepath, ...)Load calibration test data as TestDataset
load_synthetic_system(...)Generate data from the synthetic simulation

Analysis Functions

fuel_level_sensitivity(residuals)error_analysis Error statistics binned by fuel level
