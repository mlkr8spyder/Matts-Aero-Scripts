Overview

The synthetic fuel system is a 5-tank simulation of a capacitance-based fuel quantity indication (FQI) system. It models the full measurement chain from physical fuel state through probe capacitance to indicated weight, with configurable error injection for validation testing.

This page documents the simulation architecture, tank geometry, probe types, error model, and data formats. For the analysis toolkit that processes H-V table data, see the Analysis Tools Guide.

System Architecture

Measurement Chain

Physical fuel state
  → Fuel surface geometry (tank_geometry.py)
  → Probe wetted height (probe sensing model)
  → H-V table lookup (hv_table_generator.py → gauging_model.py)
  → Volume indication
  → Density × Volume = Indicated weight

Probe Types

Real Single Capacitance Probe

One physical probe measures the dielectric constant of the fuel/air column. The wetted length is converted to height, which is used for H-V table lookup. T1, T2, and T4 use this type.

Real/Pseudo Combo Dual Probe with Blend Zone

T3 uses two physical probes (lower and upper) with a blend zone from WL 90 to WL 92. Below the blend zone, only the lower probe is used. Above it, only the upper probe. In the blend zone, a weighted average transitions smoothly between them. A sinusoidal step error is injected at the blend boundaries to model real-world probe handoff discontinuities.

Pure Pseudo Projected Reading

T5 has no physical probe. Its fuel height is projected from the T3 probes using the geometric relationship between the two tanks (55" FS offset). This projection amplifies attitude errors — above 3° pitch, an additional error proportional to the lever arm is injected.

H-V Table Format

Tables are generated across a pitch/roll attitude grid (default: ±6° pitch × ±8° roll, 1° steps = 13×17 = 221 attitudes per tank).

.mat Format (for MATLAB/Simulink)

pitch_range          — [1 × 13] vector
roll_range           — [1 × 17] vector
T{n}_heights         — {13 × 17} cell array, each cell = variable-length vector
T{n}_volumes         — {13 × 17} cell array, matching heights
T{n}_cg_fs/bl/wl     — {13 × 17} cell arrays for CG coordinates
T{n}_fs_min/max, etc — scalar geometry bounds

.json Format (for Python)

{
  "pitch_range": [...],
  "roll_range": [...],
  "tanks": {
    "1": {
      "name": "Forward",
      "tables": [[{"heights_rel": [...], "volumes_in3": [...], ...}, ...], ...]
    }
  }
}

Simulation Sequences

Defuel (1000 samples)

Phase 1: T1 drains → Phase 2: T2+T4 → Phase 3: T5 → Phase 4: T3

All tanks start at 98% fill. Attitude wanders slowly around level (mean pitch 0.3°, mean roll -0.2°).

Refuel (800 samples)

Phase 1: T3 fills → Phase 2: T5 → Phase 3: T2+T4 → Phase 4: T1

All tanks start empty. Attitude wanders around level (mean pitch -0.1°, mean roll 0.3°).

Scale checkpoints are placed at phase boundaries (5 readings per sequence).

Running the System

# Generate H-V tables
python -m synthetic_fuel_system.src.hv_table_generator

# Run defuel/refuel simulations
python -m synthetic_fuel_system.src.simulate_sequences

# Generate validation plots
python -m synthetic_fuel_system.src.plot_validation

# Interactive probe coverage
python -m synthetic_fuel_system.src.probe_coverage_interactive

# Run tests
python -m pytest synthetic_fuel_system/tests/ -v
